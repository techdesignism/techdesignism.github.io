var coll = document.getElementsByClassName("collapsible"); var i; for (i = 0; i Method two: Step by step manually install & set up Besides 1-command-install, you can also install and configure manually. Step 1: Download the PHP files You’ll need the PHP Windows installer. There are a number of versions of PHP available. Make sure you get the latest PHP x64 Thread Safe ZIP package from [https://www.php.net/downloads.php](https://www.php.net/downloads.php "new tab") Please use the Thread Safe version because of the compatibility of other software, e.g. Apache. Step 2: Extract the files Create a new php folder in the root of your C:\ drive and extract the contents of the ZIP into it. PHP can be installed anywhere on your system, but you’ll need to change the paths referenced below if C:\php isn’t used. Step 3: Configure php.ini PHP’s configuration file is named php.ini This doesn’t exist initially, so copy C:\php\php.ini-development to C:\php\php.ini This default configuration provides a development setup which reports all PHP errors and warnings. There are several lines you may need to change in a text editor (use search to find the current value). In most cases, you’ll need to remove a leading semicolon (;) to uncomment a setting. First, enable any required extensions. This will depend on the libraries you want to use, but the following extensions should be suitable for most applications: extension=curl extension=openssl extension=gd extension=mbstring extension=pdo_mysql extension=pdo_sqlite extension=sqlite3 extension=fileinfo Step 4: Add C:\php to the path environment variable To ensure Windows can find the PHP executable, you need to change the PATH environment variable. Click the Windows Start button and type “environment”, then click Edit the system environment variables. Select the Advanced tab, and click the Environment Variables button. Scroll down the System variables list and click Path followed by the Edit button. Click New and add C:\php: Note that older editions of Windows have a single text box where each path is separated by a semi-colon (;). Click OK until you’re out. You shouldn’t need to reboot, but you may need to close and restart any cmd terminals you have open. Step 5: Install composer Download composer: [https://getcomposer.org/download/](https://getcomposer.org/download/ "new tab") Click " Composer-Setup.exe" on the download page. Download and run it. Step 6: Create a Laravel Project Open cmd.exe and cd to a folder you want to create a Laravel project. I use the Temp folder to demonstrate. cd %temp% Use composer to create a Laravel project called "myapp", you can replace "myapp" with any project name you like. (8.1.0 is the laravel version) composer create-project laravel/laravel=8.1.0 myapp alert-danger a { color: rgb(41, 145, 210)!important; } The Laravel version must be selected corresponding to the PHP version, for example, Laravel 8 must correspond to PHP ≥ 7.3 and ≤ 8.1. For details, please refer to [https://en.wikipedia.org/wiki/LaravelRelease_history](https://en.wikipedia.org/wiki/LaravelRelease_history "new tab") or [https://laravel.com/docs/9.x/releasessupport-policy](https://laravel.com/docs/9.x/releasessupport-policy "new tab") cd to myapp to go into the project folder, and turn it on (php artisan serve) cd myapp php artisan serve Under normal circumstances, you can see that your Laravel server is enabled, usually at 127.0.0.1:8000 You can use your web browser to view.
